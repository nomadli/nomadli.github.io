- set
    -a	标示已修改的变量，以供输出至环境变量
    -b	使被中止的后台程序立刻回报执行状态
    -d	Shell预设会用杂凑表记忆使用过的指令，以加速指令的执行。使用-d参数可取消
    -e	若指令传回值不等于0，则立即退出shell
    -f	取消使用通配符
    -h	自动记录函数的所在位置
    -k	指令所给的参数都会被视为此指令的环境变量
    -l	记录for循环的变量名称
    -m	使用监视模式
    -n	测试模式，只读取指令，而不实际执行
    -p	启动优先顺序模式
    -P	启动-P参数后，执行指令时，会以实际的文件或目录来取代符号连接
    -t	执行完随后的指令，即退出shell

## top 
01. 为当前系统运行时间 
02. up  系统运行时间
03. user  当前登录用户数
04. load average 1分钟，5分钟，15分钟 平均可执行进程百分比
05. Tasks 当前进程总数
06. running   正在运行的进程数
07. sleeping   睡眠的进程数
08. stop    停止的进程数
09. zombie   僵尸进程数
10.  Cpu(s):
    1. us  用户空间占用CPU百分比
    2. sy,  内核空间占用CPU百分比
    3. ni  用户进程空间内改变过优先级的进程占用CPU百分比
    4. id 空闲CPU
    5. wa  等待输入输出的CPU时间百分比
    6. hi    硬中断
    7. si    软中断
    8. st    实时
11.  Mem
    1. total 内存总容量
    2. used  使用的物理内存总量
    3. free  空闲内存总量
    4. buffers 用做内核缓存的内存量
13. Swap
    1. total 交换分区总量
    2. used 使用的交换分区总量
    3. free  空闲的交换分区总量
    4. cached 缓冲的交换区总量
14. PID：进程ID
15. USER: 真实用户名称
16. PR: 优先级
17. NI: Nice值，负值表示高优先级，正值表示低优先级
18. VIRT:进程使用的虚拟内存总量，单位kb VIRT=SWAP+RES
19. RES:进程使用的、未被换出的物理内存大小，单位kb RES=CODE+DATA
20. SHR:共享内存大小，单位kb
21. S:进程状态  D=不可中断的睡眠状态 R=运行 S=睡眠 T=跟踪/停止 Z=僵尸进程
22. %CPU  上次更新到现在的CPU时间占用百分比
23. %MEM  进程使用的物理内存百分比
24. TIME+  进程使用的CPU时间总计，单位1/100秒
25. COMMAND 命令名/命令行 进程名称
